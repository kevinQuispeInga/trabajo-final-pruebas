## Estructura del Repositorio y Guías de Ejecución

Cada carpeta contiene su propio proyecto independiente con configuraciones y dependencias aisladas. Haz clic en los siguientes enlaces para ver el detalle y cómo ejecutar cada tipo de prueba:

1. **[1-pruebas-unitarias](./1-pruebas-unitarias)** 🧪
   * *Tecnología:* Vitest (JavaScript)
   * *Objetivo:* Validación de lógica local de cálculos de IGV y solapamiento de fechas de reservas.
   * *Guía:* [Ver guía de ejecución de Pruebas Unitarias](./1-pruebas-unitarias/README.md)

2. **[2-pruebas-web](./2-pruebas-web)** 💻
   * *Tecnología:* Playwright
   * *Objetivo:* Pruebas de interfaz gráfica sobre DemoBlaze (navegación de laptops, adición al carrito con alertas nativas y registro de usuario).
   * *Guía:* [Ver guía de ejecución de Pruebas Web](./2-pruebas-web/README.md)

3. **[3-pruebas-api](./3-pruebas-api)** 🔌
   * *Tecnología:* Supertest + Vitest
   * *Objetivo:* Pruebas de endpoints REST y ciclo de vida de reservas sobre Restful-Booker.
   * *Guía:* [Ver guía de ejecución de Pruebas de API](./3-pruebas-api/README.md)

4. **[4-pruebas-moviles](./4-pruebas-moviles)** 📱
   * *Tecnología:* Maestro Framework (YAML)
   * *Objetivo:* Simulación de navegación, búsqueda y checkout en la aplicación Fliptronics APK.
   * *Guía:* [Ver guía de ejecución de Pruebas Móviles](./4-pruebas-moviles/README.md)

5. **[5-pruebas-rendimiento](./5-pruebas-rendimiento)** ⚡
   * *Tecnología:* Artillery
   * *Objetivo:* Pruebas de carga y estrés concurrentes sobre Restful-Booker, analizando percentiles de latencia (median, p95, p99).
   * *Guía:* [Ver guía de ejecución de Pruebas de Rendimiento](./5-pruebas-rendimiento/README.md)
